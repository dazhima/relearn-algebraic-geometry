What this is

One machine, run in four settings. The machine (Thm 7.1): giving a morphism $X \to \mathbb P^n$ is the same data as giving an invertible sheaf $\mathscr L$ on $X$ plus $n{+}1$ global sections that generate it. Maps to projective space stop being geometry you find and become algebra you specify. The four settings:

A · Morphisms to $\mathbb P^n$ — the machine itself, plus when the map is a closed immersion. B1= Thm 7.1; B2= Prop 7.3, the separates points + separates tangent vectors criterion — the actual content of "very ample."
B · Ample sheaves — $\mathscr L$ is ample if enough twisting by it makes every coherent sheaf globally generated. B3= Thm 7.6: ample $\iff$ some power very ample. Note the definition has no vanishing theorem in it — this is where the algebraic and analytic routes genuinely differ in texture.
C · Linear systems — the same machine in classical clothes: divisors in $|D|$ $\leftrightarrow$ sections mod scalars, base points = where the map isn't defined.
D · Proj, $\mathbb P(\mathscr E)$, blowing up — the machine relative to a base. B4= D8 + C12–C16: the blow-up as $\operatorname{Proj}$ of the Rees algebra, its universal property, and 7.16(c) — which cites the hinge to keep blow-ups quasi-projective.

The kill list in dependency-map.html already isolated the four items of setting A/C you must actually master; the right-hand column there showed the curve case then reduces to Riemann–Roch arithmetic you own from session 1.
